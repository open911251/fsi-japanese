# 日語陪聊 Bot 設計（Discord 掛機型）

定位：**無腳本的意義輸出練習**——像 Discord 上掛著的日本朋友。**核心目標是語音直接對話：使用者在非學習時間掛著打遊戲、看影片時，能無壓力地用日語閒聊**；文字 DM 是骨架與低強度入口。與操練工具分工：工具管句型自動化（有標準答案），bot 管真實運用（沒有標準答案）。獨立於網頁工具部署：筆電跑視窗 agent，工作站跑 bot 本體（discord.py＋Ollama），與 `server/` 同模式。

選 Discord 語音頻道當載體的理由：全螢幕遊戲時看不到文字但聽得到耳機；傳輸／降噪／與遊戲共存由 Discord 客戶端全包，筆電端零新增程式。

**本目錄只進程式與設計文件。token、對話記憶、視窗紀錄一律放 `bot/data/`（.gitignore），絕不 commit。**

## 架構

```
筆電 agent（win32gui 抓前景視窗，變化時 POST）
   │  經既有 ssh 隧道
   ▼
工作站 bot 本體（discord.py DM）
   ├─ 觸發引擎（何時開口）
   ├─ 話題選擇器（說什麼）
   ├─ 提示組裝（人設＋記憶＋語境）→ Ollama
   └─ 記憶（bot/data/：事實檔＋每日摘要＋錯誤句收集）
```

## 一、人設（自然的地基）

沒有人設的 bot 只會面試式提問，三天就乾。人設要素：

- **具體的人**：20 代後半的日本朋友（名字實作時取），有自己的工作、興趣（挑 2–3 個跟使用者有交集的：遊戲、科技、日劇）、生活節奏。**會聊自己**——單向訪談不是聊天，「俺も昨日それ見たわ」才是。
- **每日生活種子**：每天從事件池隨機抽 1–2 件「它今天發生的事」（工作被雷、看了某類節目、吃了什麼）注入 prompt，讓它主動有話講、且每天不重複。
- **語體**：タメ口＋縮約形（してる、じゃん、っけ、まじで）。禁止 assistant 腔（「何かお手伝いできますか」類句式列黑名單進 prompt）。

## 二、程度控制

- **上限 N3**：system prompt 限定文法範圍（對齊工具 28 課＝N5–N3），生詞或 N2 句型可偶爾出現，但接 Discord 劇透標籤給注釋：`難しい単語||中文意思||`——看得懂就不點，卡住才點開，不打斷對話流。
- **只說日文**：聽得懂使用者夾中文，但永遠用日文回。使用者看不懂時**換簡單說法重講**，不是切中文。
- **自適應**：使用者連續回很短／回中文／已讀不回 → 降難度、句子變短；使用者回長句 → 允許話題加深。實作＝把最近幾輪使用者回覆的長度與語言塞進 prompt 讓模型自己調。

## 三、觸發引擎（何時開口）

三類訊息來源，**回應式永遠優先，主動式嚴格節流**：

| 類型 | 觸發 | 節流 |
|---|---|---|
| 回應式 | 使用者傳訊息 | 即回，無限制 |
| 事件式 | 視窗 agent 偵測到「有趣變化」 | 全域冷卻 ≥90 分鐘；同類觸發每日 1 次；每日主動訊息上限 5 則 |
| 排程式 | 每日 1–2 個隨機時段從話題池抽題 | 併入每日上限 |

**「有趣變化」規則**（agent 端只送原始視窗資訊，判斷在 bot 端）：

- 新開遊戲／YouTube 標題變化（換影片）→ 好奇或吐槽
- 同一編輯器視窗連續 2 小時以上 → 關心一句（「まだやってんの？」）
- 23:30 後仍在工作類視窗 → 深夜吐槽
- 當天第一次有視窗活動 → 可當開場（不做固定「おはよう」排程，太機器人）

**防煩機制（比觸發本身重要）**：

- 靜音時段（睡眠、可設定）；會議／全螢幕應用偵測到就閉嘴
- **退避**：上一則主動訊息沒得到回覆 → 主動頻率減半，連續兩則沒回就當天閉嘴；使用者一回覆即重置
- 銀行、私人通訊等視窗類別列入忽略名單，agent 端就過濾掉不上傳

## 四、話題選擇器（趣味的來源）

主動訊息按權重混合，避免單一模式膩掉：

| 權重 | 來源 | 例 |
|---|---|---|
| 40% | 視窗語境 | 「そのゲームまだクリアしてないの？笑」——像朋友瞄到，**禁止監視感描述**（「見てるよ」類） |
| 25% | 記憶追蹤 | 昨天摘要裡的未完話題：「そういえば面接どうだった？」——黏性最高的一類 |
| 20% | 人設自身 | 每日生活種子：「今日残業でさ…聞いてくれる？」 |
| 15% | SRS 句型（二期） | 讀工具匯出的今日操練句型，設計一個自然會用到該句型的問題（不點破） |

## 五、回覆風格（自然的細節）

全部寫進 system prompt：

1. 每則 1–3 短行，Discord 風；不用條列、不寫作文
2. **每則最多一個問句**，且允許零問句（「まじか笑」純反應）——連環提問＝面試，最快殺死聊天的方式
3. 長度跟隨使用者：對方回三個字，你也短
4. 偶爾拆成連續兩則短訊（模擬真人打字節奏，機率觸發）
5. 顏文字／emoji 低頻使用，同一個不連用

## 六、糾錯策略

- **預設 recast**：使用者說錯，回覆中自然用正確形式重述一次，不點破、不中斷
- 同一錯誤第三次出現 → 回覆末尾加一行劇透標籤小注：`||〜たら の方が自然かも||`
- 使用者打「添削して」→ 對最近幾句逐句批改（唯一的老師模式入口）
- **錯誤句全部寫進 `bot/data/errors.jsonl`**（原句＋修正＋日期）——定期貼回工具「自訂教材」操練，形成 bot→工具 的回流閉環

## 七、記憶

規模小，檔案就夠，不上向量庫：

- **短期**：最近 N 輪對話直接進 prompt
- **事實檔** `facts.md`：使用者固定資料（工作、目標、喜好），LLM 從對話中發現新事實就追加
- **每日摘要** `summary/YYYY-MM-DD.md`：每天清晨把前日對話摘要成 5 行內＋「未完話題」清單；prompt 注入＝事實檔＋最近 2 天摘要
- 未完話題被追問過就標記關閉，避免重複問

## 八、提示組裝（兩種模式）

```
[system] 人設＋語體規則＋程度上限＋風格規則＋糾錯規則
[context] 現在時刻｜視窗語境（若有）｜事實檔｜最近摘要｜今日生活種子｜（二期）今日 SRS 句型
[history] 最近 N 輪
[task] 回應式：回覆使用者
       主動式：「基於觸發原因 X 產生一則自然的開場訊息（禁止提及你能看到畫面這件事的機制）」
```

模型：工作站上較大的那顆（タメ口自然度需要）；temperature 偏高（~0.9）；閒聊頻率低，延遲不敏感。

## 九、先生モード（雙模式）

打「先生モード」切換、「友達モード」切回，記憶（事實檔、摘要、歷史）兩模式共用。兩模式分工：

| | 友達モード（預設） | 先生モード |
|---|---|---|
| 語體 | タメ口＋縮約形 | 丁寧体（です・ます）——職場敬語的活教材 |
| 主導權 | 對方有空才聊 | bot 主導：按「今日の練習テーマ」一次一問、引導展開 |
| 糾錯 | recast 不點破 | 每句先一行直改（「→ 〜の方が自然です」）再繼續對話 |
| 視窗觸發 | 全開 | 關（老師不偷看畫面）；排程時段變成練習邀請 |
| 主題 | 視窗／記憶／生活種子 | 練習主題池（生活場景：市役所、面接、電話応対…與 ROADMAP 生活場景課同一套） |

切到先生モード時立即出第一題；每日重置會換主題。節流參數兩模式共用（老師也不煩人）。

## 十、語音模式（核心目標）

管線（全在工作站）。**收發音走 Node 中繼**：Discord 2026-03 起強制語音 E2EE（DAVE），Python 生態（discord-ext-voice-recv）解不開收音（實測只收到一個亂碼封包），discord.js `@discordjs/voice` 0.19.2+ 配 `@snazzah/davey` 已修復——故耳朵嘴巴用 Node、大腦留 Python：

```
語音頻道 ⇄ relay/relay.js（discord.js＋DAVE：收音、AfterSilence 800ms 斷句、播放、barge-in）
              │ PCM／WAV（127.0.0.1:8789 /voice/utterance、/voice/greet；控制 8790 /join /leave）
              ▼
         bot.py（大腦）→ faster-whisper（已架）→ Ollama（同一套人設＋記憶）→ VOICEVOX → WAV 回 relay
```

- relay 與 bot.py 用同一個 token 各開一條 gateway session（Discord 允許）；DM 指令仍由 bot.py 處理，「おいで」轉發控制指令給 relay
- 延遲預估 2–4 秒；優化手段＝LLM 逐句 stream、逐句餵 TTS，體感壓到 1–2 秒

**語音的無壓力對話規則（與文字模式相反的部分）**：

1. **沉默是正常狀態**：絕不填補沉默、絕不問「還在嗎」；使用者開口才接話
2. 回應 1–2 句、以反應為主（「あー、それな」）；問句頻率比文字更低（對方在打遊戲）
3. **語音中完全不糾錯**：STT 文字默默進 errors.jsonl，嘴上絕不提；糾錯留給文字「添削して」
4. 主動開口每小時最多 1–2 句，只用視窗事件當引子；被晾著就閉嘴（沿用退避）
5. **barge-in**：使用者開口立即停止播放，不搶話
6. VOICEVOX 語速先調慢一檔，可指令調整

## 十一、分期

1. **v1（已完成，待部署）**：文字 DM——視窗 agent＋事件/排程觸發＋人設聊天＋recast＋檔案記憶＋先生モード。是語音層的骨架：人設、記憶、觸發、Ollama 串接全數複用，並先驗證本地模型口語自然度
2. **v2：語音模式**（見第十節）——核心目標，v1 驗證通過即做
3. **v3**：SRS 句型注入（工具加匯出）＋錯誤句回流自訂教材的順手流程
